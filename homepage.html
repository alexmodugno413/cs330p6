Assistive device for arthritis - created for an Engineering Design class at Northwestern

Problem the device addressed

My client, an elderly woman, was diagnosed with arthritis, which caused pain, swelling, and difficulty moving the joint. The client’s hands were disfigured and she found difficulty forming a closed fist, leading to an issue of reduced grip strength. She wanted to improve her ability to grip on cylindrical handles of varying diameter.

Methodology to develop design

The design our team created is called the adjustable grip mold. It is an assistive device consisting of two major components: (1) sponge rubber material padding and (2) a cotton glove with velcro fingertips.

The rubber padding is composed of a closed cell neoprene sponge foam rubber sheet. It provides friction to objects/handles and is waterproof, adjustable, flexible, and strong.

The glove is a black cotton jewelry inspection glove containing velcro dots on the fingertips that attach to the padding to allow the user more control while moving the padding. The padding reduces the user’s pain since it decreases the amount of grip needed to grab handles/objects.

Design search algorithms to find the best way to travel between different places on a map - created for an Intro to AI class at Northwestern

Goal of the assignment

Create an implementation of Breadth-first Search (BFS), Depth-first Search (DFS), and the A* algorithm that all find the best way to travel between any pair of landmarks in a town. A comparative analysis of all three is done to select the best one.

Details about implementation

The BFS and DFS algorithms are given a time map, start location, and end location as input. The time map specifies the expected time it takes a driver to go from one landmark to a neigbhoring landmark. This map is in the form of a graph, and BFS and DFS are tree search algorithms operating on the graphs that the maps form.

The A* algorithm is given a time map and distance map, which is the straight line distance between two points, as input. The A* algorithm uses these distances as a heuristic (how far one is from the destination). This means the distance from the start to your current location will be time-based, but the distance from your current location to the destination will be distance-based.

The result that is returned is a list of strings (locations) that represent the fastest route between one location and another.
